# Security Policy

## Supported versions

This project is pre-release. Security fixes target the latest commit on `main`.

## Reporting a vulnerability

Please open a private security advisory on GitHub if available, or contact the maintainer through the repository owner profile.

Do not include secrets or private audience datasets in public issues.

## Data handling expectations

- Comment imports may contain personal data.
- Do not commit real imports.
- Keep source traceability without exposing more data than needed.
- Prefer local processing for early MVP validation.
