# Contributing

This project is early-stage. Contributions should keep the MVP narrow and evidence-driven.

## Workflow

1. Open or claim a Beads issue.
2. Make one logical change.
3. Run validation locally.
4. Commit with a Conventional Commit message.
5. Open a pull request with the test plan and public-safe evidence.

## Public-safety rules

Do not commit:

- API keys, tokens, cookies, or OAuth data.
- Private comment exports or personal audience data.
- Private filesystem paths, hostnames, or infrastructure details.

Use synthetic data or short public excerpts with links.
